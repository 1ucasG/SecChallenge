Tutoriel GraphQL

Qu’est-ce que GraphQL ?

GraphQL est un langage de requête pour les API créé par Facebook en 2012 et devenu opensource en 2015.

GraphQL utilise un schéma pour décrire les types de données disponibles dans l’API, ainsi que les relations entre ces types. Les requêtes GraphQL peuvent également inclure des arguments pour filtrer, trier ou paginer les données.

En résumé, GraphQL est une technologie de requête flexible et puissante qui permet aux clients de récupérer des données spécifiques à partir d’une API de manière efficace et contrôlée.

Quelle est la différence avec les API REST ?

La principale différence entre GraphQL et REST est que REST est basé sur une architecture de ressources, où chaque endpoint correspond à une ressource. Voici des exemples d’endpoints REST : - /users - /articles - /login

GraphQL est basé sur une architecture de requête, où les clients spécifient exactement les données qu’ils souhaitent récupérer.

Dans GraphQL, il n’y a qu’un seul endpoint, par exemple /graphql, on peut spécifier exactement les données que l’on veut récupérer, ce qui évite l’under/over fetching des APIs REST. De plus, le typage fort des données permet d’avoir un code maintenable et une API facile à consommée côté client.

Les bases de GraphQL

Lorsque l’on crée une API GraphQL, on crée des schémas et des types qui vont typer toutes les données de l’API.

Par exemple, on peut créer un schéma User qui contient les champs email et password.

Il y a deux types de requêtes GraphQL, les query et les mutation. Nous pouvons les comparer aux méthodes GET et POST que l’on utilisera dans une API classique REST. Les query nous permettent de récupérer des données, tandis que les mutation nous permettent d’update/insérer des données.

Les query et mutation prennent éventuellement des arguments, si on a besoin de transmettre des données ou de récupérer une donnée précise.

Par exemple, pour récupérer un User précis, on peut créer une query qui prends en paramètre l’id de l’utilisateur et renvoie l’utilisateur avec cet id. La personne qui fait la requête peut spécifier les champs qu’il veut recevoir, par exemple s’il n’a besoin que de l’email de l’utilisateur, sa requête ressemblerait à : 

 query {
    user(id: 1) {
    email
    }
    }

Plutôt que :

 query {
    user(id: 1) {
    id
    email
    password
    }
    }
