LOG4SHELL

Log4Shell, Ã©galement connue par son numéro CVE-2021-44228, était une vulnérabilité zero-day exploitée par exécution de code arbitraire et touchant l'utilitaire Java Log4j.

La vulnérabilité tire parti du fait que Log4j ne vérifie pas les requêtes LDAP (Lightweight Directory Access Protocol) et JNDI (Java Naming and Directory Interface). Cela permet aux attaquants d'exécuter du code Java arbitraire sur un serveur ou un autre ordinateur. Les services concernés incluent iCloud, l'édition Java de Minecraft et Steam.

Elle est considéré comme la vulnérabilité la plus importante et la plus critique de la dernière décennie par la directrice de la Cybersecurity and Infrastructure Security Agency (CISA). L'Apache Software Foundation, dont Log4j est un projet, a attribué à Log4Shell une note CVSS (Common Vulnerability Scoring System) de 10, la note la plus élevée possible.

Log4j est un framework de journalisation open source qui permet aux développeurs de logiciels d'enregistrer diverses données au sein de leur application. Ces données peuvent également inclure une entrée utilisateur. Il est utilisé de manière omniprésente dans les applications Java, en particulier les logiciels d'entreprise. La bibliothèque est ainsi utilisée dans des logiciels de bureautique, dans des serveurs ou des systèmes embarqués comme dans certaines voitures. L'exploitation de la vulnérabilité CVE-2021-44228 passe par l'insertion d'un appel JNDI dans une requête. Ces appels sont utilisés pour rédiger des journaux de manière dynamique, par exemple pour insérer dans les entrées du journal les résultats d'un calcul complexe géré par un système tiers. Si la requête est enregistrée dans les journaux gérés par une version vulnérable de Log4J, alors la vulnérabilité se déclenche.

La CVE-2021-44228 peut être utilisée avec des charges utiles qui génèreront respectivement des appels via le protocole LDAP ou DNS, vers un serveur contrôlé par un tiers. Un pirate peut aussi utiliser ce mécanisme pour provoquer une exfiltration de données, via un appel à des variables Java comme les variables de l'environnement utilisateur. Une attaque peut facilement être réalisée à partir d'un service d'écoute DNS, comme DNSLog, puis une charge utile est créée qui génèrera un appel vers ce service d'écoute. Afin de maximiser les chances de réussite de l'attaque, les pirates insèreront la charge utile dans des éléments classiquement enregistrés comme les entêtes HTTP.

Les correctifs de cette vulnérabilité ont été rendus disponibles le 6 décembre 2021, trois jours avant la publication de la vulnérabilité. Le correctif inclut la restriction des serveurs et des protocoles pouvant être utilisés pour les recherches, qui peuvent être configurés à l'aide de plusieurs propriétés système. Les administrateurs système ont été invités à évaluer la situation et à mettre en place des mesures d'atténuation le plus rapidement possible, soit en mettant à jour la bibliothèque logicielle, soit en désactivant les recherches à l'aide de la propriété système.
