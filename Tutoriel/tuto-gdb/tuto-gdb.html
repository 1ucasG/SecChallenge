GNU Debugger

1. PRÉSENTATION DE L'OUTIL

GNU Debugger ou GDB est un debugger développé à l'origine pour le projet GNU mais fonctionnant sous plusieurs systèmes UNIX. GDB fonctionne pour plusieurs langages de programmation tels que C, C++ ou Fortran.

GDB est utilisé pour débugger un programme ligne par ligne durant son exécution ou post mortem quand le programme a été arrété suite à une erreur. Son interface est entièrement en lignes de commande bien que certaines applications permettent de l'utiliser avec une interface graphique. Certains IDE tels qu'Eclipse l'utilisent directement en arrière-plan lors de l'exécution d'un programme.

2. UTILISATION

Pour utiliser GDB, il faut que votre programme possède des informations de debug, pour cela, il suffit d'ajouter l'argument -g lors de la compilation : gcc -g programme.c -o programme
Il reste ensuite à taper la commande gdb programme afin de lancer le debug.

Une fois le debug en cours, de nombreuses commandes permettent de l'utiliser :

- break programme.c:x ou b programme.c:x permet de placer un breakpoint à la ligne x du programme "programme.c".
- delete x ou d x permet de supprimer le breakpoint à la ligne x (ou tous les breakpoints si aucun argument n'est ajouté).
- info breakpoints liste tous les breakpoints du programme.
- run ou r permet de lancer le programme jusqu'au prochain breakpoint.
- continue ou c permet de relancer un programme en pause jusqu'au prochain breakpoint.
- next ou n exécute la prochaine instruction du programme mais ne rentre pas dans les fonctions (avec un nombre en argument, il est possible d'exécuter plusieurs instructions).
- step ou s exécute la prochaine instruction du programme et peut rentrer dans les fonctions.
- finish ou f exécute le programme jusqu'à sortir de la fonction courante.
- print variable permet d'afficher la valeur de la variable "variable" lorsque le programme est arrété.
